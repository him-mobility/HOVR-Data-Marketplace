# HOVR — 로봇이 모은 도시 현장 데이터 마켓플레이스 (by HIM)

> **데이터를 SQL이 아니라 자연어로 탐색하다.**

HOVR은 전국 주요 도시(광주·수도권·부산·대구·대전·울산·세종 등)에서 로봇이 직접 주행하며 수집한
약 **9.8만 건**의 도시 현장 데이터(위치·현장 상황·도로 이벤트·사진/영상)를 **구독·맞춤**으로 제공하는
데이터 마켓플레이스입니다.

전문 지식이나 SQL을 몰라도 됩니다. 화면에서 **자연어로 물어보면** 원하는 지역·기간·상황을 골라
지도·차트·표로 미리 보고, 비즈니스에 맞는 데이터를 **AI 추천**으로 받아볼 수 있습니다.

---

## 이런 데이터를 다룹니다

로봇이 도시를 주행하며 모은 현장 데이터를 4가지 형태로 제공하며, 모두 **위치를 기준으로 하나로 연결**됩니다.

| 구분 | 데이터 | 내용 |
|------|--------|------|
| A | **위치 기록** | 로봇이 주행한 도로 위 위치·이동 정보 (모든 데이터의 기준) |
| B | **현장 관측** | 각 지점의 도로·교통·날씨 등 현장 상황 |
| C | **도로 이벤트** | 포트홀·불법주정차·침수·사고 등 7종으로 분류된 도로 상황 |
| D | **현장 사진·영상** | 현장을 담은 이미지/영상 (개인정보는 가려진 상태로 제공) |

도로 이벤트 7종: **불법주정차 · 포트홀 · 정차차량 · 공사 · 인파밀집 · 침수 · 사고**

## 화면 이동 — 어디를 클릭하면 되나요

모든 페이지 위쪽에는 **상단 메뉴 바**가 고정되어 있습니다. 여기서 대부분의 화면으로 이동할 수 있습니다.

- 맨 왼쪽 **HIM 로고** → 언제든 **홈**으로 돌아갑니다.
- 메뉴: **「데이터 상품」 · 「데이터 구성」 · 「요금」** → 홈의 해당 섹션으로 스크롤됩니다.
- ✨ 표시된 청록색 버튼 **「맞춤형 데이터」** → **AI 추천** 화면으로 이동합니다.
- 맨 오른쪽 짙은색 버튼 **「표본 데이터 둘러보기 →」** → **데이터 대시보드(Explorer)**로 이동합니다.
- 화면 **우측 하단의 동그란 채팅 버튼** → **Q&A 챗봇**이 열립니다. (대시보드 화면 제외, 어디서나)

## 페이지 안내 — 무엇을 할 수 있나요

### 🏠 홈
제품의 첫 화면입니다. 어떤 데이터를 받아볼 수 있는지, 데이터가 어떻게 구성·연결되는지,
그리고 요금제를 한눈에 볼 수 있습니다.

- **데이터 상품** — 위치·관측·이벤트·사진 4종 데이터셋 소개
- **데이터 구성** — 데이터가 위치를 기준으로 서로 연결되는 방식
- **요금** — 무료 체험 / 서비스 구독($100/월) / 맞춤·대량(견적) 3가지 플랜
- 첫 화면(히어로)의 **「나만의 맞춤형 데이터」** 버튼 → AI 추천으로, **「요금 보기」** 버튼 → 요금 섹션으로 이동합니다.
- 요금의 **무료 체험** 카드에서 **「표본 데이터 둘러보기」**를 누르면 대시보드로 이동합니다.

### 🔎 AI 추천 — 맞춤형 데이터
상단 메뉴의 ✨**「맞춤형 데이터」** 버튼으로 들어갑니다.
"내 일에 맞는 데이터가 뭔지 모르겠다"면 여기서 시작하세요. 몇 가지만 고르면 AI가
보유 데이터 중 가장 잘 맞는 데이터셋을 **이유와 함께** 추천합니다.

1. **직업** 선택 (예: 보험·금융, 물류·모빌리티, 도로·시설 관리 …)
2. **활용 목적** 선택 (위험 구간 분석, 리스크·보험 산정, AI 모델 학습 …)
3. **관심 지역** 선택 (전체 또는 광주·수도권·부산 등)
4. **관심 항목** 선택 (포트홀·침수·사고 등 도로 상황)
5. (선택) **세부 내용**을 자유롭게 입력
6. **「AI 추천 받기」** 버튼 → 추천 데이터셋 카드가 표시됩니다.

각 추천 카드에서 **「지도에서 살펴보기」** 버튼을 누르면 해당 조건이 그대로 적용된 대시보드로 이동하고,
**「구독 문의」** 버튼으로 바로 도입 문의를 보낼 수 있습니다.

### 🗺️ 데이터 대시보드 — Explorer
상단 메뉴 맨 오른쪽 **「표본 데이터 둘러보기 →」** 버튼, 또는 요금의 무료 체험 카드로 들어갑니다.
실제 데이터를 지도·차트·표로 둘러보고, **자연어로 탐색**하는 핵심 화면입니다. (무료 체험)

- 상단의 **「지도」 · 「차트」 · 「데이터 표」** 버튼으로 보고 싶은 패널을 켜고 끌 수 있습니다.
  - **지도** — 데이터 포인트를 지도 위에 표시하고, 보고 있는 영역의 건수와 최다 발생 구간(핫스팟)을 알려 줍니다.
  - **차트** — 이벤트 종류별 분포, 추세 등을 시각화합니다.
  - **데이터 표** — 개별 기록을 표로 확인하고, **줄을 클릭하면** 상세 정보 패널이 열립니다.
- **「필터」** 버튼 → 지역·이벤트 종류·도로명·기간으로 데이터를 좁힐 수 있습니다.
  적용된 필터는 상단에 토큰으로 표시되며, **토큰을 클릭하면** 바로 제거됩니다.
- **HOVI 에이전트(우측 패널)** — 예: *"광주에서 비 올 때 사고 많은 구간 보여줘"* 처럼
  자연어로 물어보면, AI가 데이터를 분석해 답하고 **대시보드 화면을 그 조건에 맞게 자동으로 맞춰** 줍니다.

> AI 추천 카드의 **「지도에서 살펴보기」**로 들어오면, 추천 조건이 처음부터 적용된 상태로 열립니다.

## 자주 쓰는 흐름

- **데이터가 어떤 건지 빠르게 보고 싶다** → 상단 **「표본 데이터 둘러보기」** → 대시보드에서 무료로 탐색
- **내 업무에 맞는 데이터를 찾고 싶다** → 상단 ✨**「맞춤형 데이터」** → AI 추천 → 카드의 **「지도에서 살펴보기」**
- **궁금한 점(요금·범위·도입)을 묻고 싶다** → 어느 페이지든 **우측 하단 채팅 버튼**으로 Q&A
- **특정 조건의 데이터를 분석하고 싶다** → 대시보드 우측 **HOVI 에이전트**에 자연어로 질문

## 개발자용 — 로컬 실행

Next.js(App Router) 기반이며, 데이터는 로컬 SQLite에 시드(seed)됩니다.
AI 기능(추천·에이전트·Q&A)은 `ANTHROPIC_API_KEY` 환경 변수로 동작하며,
키가 없으면 미리 준비된 시나리오/규칙 기반 응답으로 자연스럽게 대체됩니다.

```bash
# 1) 의존성 설치
npm install

# 2) 데이터 시드 (data/robot-data-lake.sqlite 생성)
npm run seed

# 3) 개발 서버 실행 → http://localhost:3000
npm run dev
```

```bash
# 테스트 (시드 후 vitest 실행)
npm test

# 프로덕션 빌드 / 실행
npm run build
npm start
```

### 환경 변수 (`.env.local`)

| 변수 | 설명 |
|------|------|
| `ANTHROPIC_API_KEY` | AI 추천·HOVI 에이전트·Q&A 챗봇 사용 시 필요 (없으면 대체 응답) |
| `AGENT_MODEL` | 사용할 Claude 모델 (기본: `claude-sonnet-4-6`) |

### 기술 구성

- **프레임워크**: Next.js 14 (App Router), React 18, TypeScript
- **스타일**: Tailwind CSS
- **데이터**: better-sqlite3 (읽기 전용 연결로 서비스)
- **지도/차트**: MapLibre GL, Recharts
- **AI**: Anthropic Claude SDK (도구 호출 기반 에이전트 + 지식베이스 Q&A)

### 주요 API

| 경로 | 역할 |
|------|------|
| `/api/geo` `/api/facets` `/api/aggregate` | 대시보드 지도·필터·집계 데이터 |
| `/api/records` `/api/record/[idx]` | 데이터 표 목록 / 상세 |
| `/api/recommend` | 직업·목적 기반 AI 데이터셋 추천 |
| `/api/agent` | HOVI 자연어 분석 에이전트 (대시보드 동기화) |
| `/api/chat` | 지식베이스 기반 Q&A 챗봇 |

> 요청 폭주를 막기 위한 레이트 리밋(`src/lib/ratelimit.ts`)과, AI 응답에서 내부 구현 용어가
> 노출되지 않도록 하는 "고객 표현 위생(customer hygiene)" 규칙이 시스템 프롬프트에 적용되어 있습니다.
